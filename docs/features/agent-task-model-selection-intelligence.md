Right now Git Loopy only asks the user upon running the Git Loopy initialization on first run and is editable through the config.yaml where the user can change what model and reasoning level it should use which means Git Loopy is centered on single model and reasoning choice. Choice that, once sent in the config file, applies to all issues that are in scope for Git Loopy to complete. We should add support and guided assistance to the user to specify what kind of model to use and the reasoning efforts for various tasks, that an issue may be comprised of. For example, planning tasks probably make more sense to use a very powerful model like Claudopus 4.8 with max reasoning or Fable 5, or GPT 5.6/Sol Max Reasoning, but it probably doesn't make sense to use those models for something simple like writing markdown files or updating details on GitHub issues, etc. We should try to support very common, core, broad, generalized tasks to model types that are best suited for those activities to optimize costs and also the speed of completion of various activities and the loops that get loopy will loop through for us. A user should be able to set a global model that could be used to perform every single task, no matter what kind of task and issue it is, or have the option to specify what models to use for specific types of issues and tasks.
