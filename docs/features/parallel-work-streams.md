Right now, the core of this project focuses on doing only one task at a time, no exceptions when a user runs the GitLab Orchestrator Framework. I think it's time that we look at adding some kind of workstream parallelization where GitLab Orchestrator intelligently works on as many issues as possible within the current framework of slices, etcetera. It is absolutely critical that the core of this product does not change the way that it works on issues and how issues relate to each other through their respective slices, and that other agents never have the opportunity to interfere with another agent that's working at the same time in parallel. The goal here is just to increase efficiencies without sacrificing or introducing additional risk to the workflows.
